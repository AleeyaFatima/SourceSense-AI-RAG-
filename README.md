# SourceSense AI — Enterprise Knowledge Copilot with Verified Citations

SourceSense AI is a production-grade Retrieval-Augmented Generation (RAG) search pipeline engineered to demonstrate robust semantic-lexical hybrid search, citation verification, and full retrieval transparency. The project balances search precision and context faithfulness, backed by an interactive dashboard built on top of Next.js 15 and FastAPI.

## Key Design Decisions

### 1. Hybrid Retrieval (Dense Semantic + Sparse Lexical)
Pure vector search often misses specific terminology, configuration codes, or unique serial numbers (e.g. "AES-GCM-256"). Combining Qdrant dense vector cosine similarity with BM25 word-frequency indexing ensures both conceptual semantics and exact matches are captured.

### 2. Reciprocal Rank Fusion (RRF) with $k=60$
Since dense and sparse algorithms produce scores in different ranges (cosine similarities are 0-1, whereas BM25 scores can scale arbitrarily), direct score addition fails. RRF bypasses score calibration by evaluating the *ranks* of retrieved candidates:
$$RRF(d) = \sum_{m \in M} \frac{1}{60 + rank_m(d)}$$

### 3. Citation Verification Engine
To combat LLM hallucinations, SourceSense runs a claims verification pass. For each citation token (e.g. `[1]`) generated by the LLM, the engine isolates the sentence, performs token-level Jaccard overlap scoring against the cited source chunk, and classifies its support status:
*   **Verified** ($\ge 70\%$ score): Confirmed statement integrity.
*   **Warning** ($30\% - 70\%$ score): Partial match, user warning flagged.
*   **Failed** ($< 30\%$ score): Low word overlap, flagged as potential hallucination.

### 4. RAGAS Quality Metrics
The analytics dashboard tracks key pipeline health vectors using automated evaluation criteria:
*   **Retrieval Precision**: Relevance of retrieved source text chunks to the query.
*   **Faithfulness**: Degree to which the LLM response relies only on the retrieved context.
*   **Answer Relevance**: Measure of how direct the answer is to the user request.
*   **Citation Accuracy**: Ratio of verified citations vs. total citations.

## Setup & Execution

### Prerequisites
*   Node.js (v18+)
*   Python (3.10+)

### 1. Start Backend FastAPI Server
First-time startup automatically indexes three pre-loaded sample documents (SLA guidelines, Security Standards, RAG architecture documentation) into SQLite and local Qdrant within seconds!

```bash
cd backend
# Create virtual environment
python -m venv .venv
# Activate virtual environment
.venv\Scripts\activate   # Windows Powershell
source .venv/bin/activate # Unix/macOS
# Install dependencies
pip install -r requirements.txt
# Run FastAPI server
python main.py
```

FastAPI server runs on `http://localhost:8000`. API Swagger documentation is available at `http://localhost:8000/docs`.

### 2. Start Frontend Next.js Client
```bash
cd frontend
# Install packages
npm install
# Run Next.js in development mode
npm run dev
```

Next.js application runs on `http://localhost:3000`.

## Environment Configuration (`.env`)

To use cloud services, create a `.env` file in `/backend`:
```env
# Database Settings (Fallback: local SQLite sourcesense.db)
DATABASE_URL=postgresql://user:pass@neon-host/db

# Vector DB Settings (Fallback: local qdrant_db folder)
QDRANT_URL=https://your-qdrant-cluster.aws.qdrant.io
QDRANT_API_KEY=your-api-key

# LLM Providers (Fallback: local streaming simulation)
OPENAI_API_KEY=your-openai-api-key
```
